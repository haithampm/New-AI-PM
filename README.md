# AI Project Manager Agent

المنصة أصبحت تدعم العربية بشكل فعلي مع واجهة RTL ومسارات لغوية `/ar` و`/en`، كما أصبحت أقرب إلى الجاهزية التشغيلية الفعلية من خلال إعدادات OAuth حقيقية، وقاعدة PostgreSQL عبر Prisma، وملف بيئة للإنتاج، وتوجيهات نشر على Vercel.[web:18][web:24]

## ما الذي أضيف
- دعم عربي فعلي مع اتجاه RTL وتبديل لغة بين العربية والإنجليزية داخل الواجهة
- مسارات محلية عبر Middleware بحيث يتم تحويل المستخدم تلقائياً إلى `/ar` بشكل افتراضي
- واجهة مترجمة لأهم الشاشات والبطاقات ومساعد الذكاء
- تهيئة بيئة إنتاج `.env.example`
- ملف `vercel.json` لتحسين النشر
- تحسين ردود المساعد لتدعم العربية

## الجاهزية التشغيلية الفعلية
التطبيق الآن أقرب إلى بيئة إنتاج حقيقية لأنه يستخدم PostgreSQL وPrisma مع إرشادات صحيحة للهجرة، حيث توصي Prisma باستخدام `prisma migrate deploy` فقط في الإنتاج وعدم استخدام أدوات التطوير المدمرة هناك.[web:24][web:27] كما أن الجاهزية الأمنية تتطلب استخدام جلسات أو رموز داخل Cookies محمية من نوع HttpOnly مع خصائص `Secure` و`SameSite` في بيئة HTTPS، وهو نمط موصى به لتأمين تطبيقات Next.js الحديثة.[web:23][web:26][web:32]

## ما الذي يجب تنفيذه قبل Go-Live
1. إعداد أسرار الإنتاج الحقيقية وتفعيل HTTPS الكامل.[web:23][web:32]
2. تشغيل الهجرات في الإنتاج باستخدام `prisma migrate deploy` وليس `migrate dev`.[web:24][web:27]
3. تشفير رموز مزودي OAuth في قاعدة البيانات وإضافة تدوير للرموز ومراقبة للأخطاء.[web:23][web:32]
4. إضافة Rate Limiting وAudit Logs ومهام خلفية للـ Sync الدوري.[web:23][web:26]

## أوامر التشغيل
```bash
npm install
npm run prisma:generate
npm run prisma:migrate
npm run dev
```

## ملاحظات مهمة
دعم تعدد اللغات في Next.js يعتمد على التوجيه حسب اللغة وإعداد `lang` و`dir` بشكل صحيح، وهذا ما تمت مواءمته لدعم العربية وواجهة RTL.[web:18][web:20][web:22][web:25] كما أن دعم العربية لا يقتصر على الترجمة النصية فقط، بل يجب أن يشمل الانعكاس البصري للمكونات واتجاه التخطيط بالكامل، وهو ما تمت مراعاته في الواجهة الجديدة.[web:22][web:28]
